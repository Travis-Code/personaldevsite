About Me

I'm a software engineer with a background in technical business analysis and application support, contributing directly to compliance-driven public sector systems.

My work spanned both the technical and human sides of software—supporting backend workflows through SQL-based reporting, REST API testing, and application troubleshooting, while also presenting findings to clients to understand how systems behaved in real use. Seeing where requirements met real behavior is what pushed me toward more hands-on engineering work.

My work now centers primarily on JavaScript and TypeScript, building full-stack, database-driven applications. Alongside this, I've been actively developing game projects using Godot and Phaser, exploring systems design, state management, and performance-aware gameplay logic. I also have supporting experience with Python and a foundational background in Java from earlier academic work.

Across all of it, I bring a disciplined, engineering-first approach shaped by time spent in regulated environments—focused on clarity, reliability, and learning by building.
